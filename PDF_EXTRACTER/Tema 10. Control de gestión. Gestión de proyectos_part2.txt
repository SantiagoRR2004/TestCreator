Una red PERT es un diagrama de flujo que representa la secuencia de actividades necesarias para completar un proyecto y el tiempo o los costes asociados con cada actividad. PERT facilita la comparación de los efectos que las acciones alternativas tendrán en la programación y en los costes, y permite a los gerentes monitorizar el progreso de un proyecto, identificar posibles cuellos de botella y cambiar los recursos según sea necesario para finalizar el proyecto a tiempo. Para construir una red PERT se necesita conocer tres elementos: eventos, actividades y ruta crítica. • Los eventos son puntos finales que representan la finalización de actividades principales. Los eventos (o hitos) indican que un componente importante ha terminado. En la red PERT se usan círculos para indicar eventos. • Las actividades son las acciones que tienen lugar. El tiempo que consume una actividad está determinado por el tiempo o los recursos necesarios para progresar de un evento a otro. En la red PERT se usan flechas para representar actividades. • La ruta crítica es la secuencia más larga o que consume más tiempo de eventos y actividades necesarias para completar el proyecto en el menor tiempo posible. Desarrollo de una red PERT El desarrollo de una red PERT se puede traducir en cinco pasos: 1. Identificar todas las actividades claves necesarias para completar el proyecto. El cumplimiento de cada actividad genera un conjunto de eventos o resultados. 2. Determinar el orden en que deben completarse estos eventos, o el orden de dependencia. 3. Representar en un diagrama el flujo de actividades desde el inicio al fin, identificando cada actividad y su relación con todas las demás actividades. El resultado es un diagrama de flujo denominado red PERT. 4. Calcular el tiempo estimado para completar cada actividad utilizando un promedio ponderado que considere una estimación de tiempo optimista (t ) o cuánto tiempo tardaría en concluir la actividad en condiciones ideales, una estimación más probable (t ), el tiempo que la actividad que normalmente m debería tardar la actividad, y una estimación pesimista (t ) que representa el tiempo que tardaría una actividad en las peores condiciones posibles. Utilizando la red PERT, que contiene las estimaciones de tiempo para cada actividad, se puede determinar un programa para las fechas de inicio y finalización de cada actividad y para todo el proyecto. Cualquier retraso que ocurra a lo largo de la ruta crítica requiere la mayor atención porque retrasa todo el proyecto. Es decir, el camino crítico no tiene holgura, ya que cualquier retraso se traduce en un retraso en el plazo final de ejecución del proyecto.
